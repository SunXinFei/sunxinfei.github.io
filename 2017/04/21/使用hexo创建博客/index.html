使用Hexo创建静态博客

发表于 2017-04-21 | 分类于 博客 | | 阅读次数

很多人有写博客记录的习惯，一般的博客平台自由度不够高，也有很多人对于搭建和维护服务器很头疼，所以就诞生出静态博客。静态博客顾名思义，博客里面的内容都是静态页，而且现在静态博客发展很成熟，插件也非常的丰富，满足我们大部分的博客需要了。搭建的工具有很多，比较出名的是Jekyll和Hexo，这里我们使用Hexo搭建静态博客。

本地环境配置

Win10
Git 下载Git并执行即可完成安装。
NodeJS: NodeJS6.10.2 下载NodeJS并执行即可完成安装。
Hexo : Hexo3.3.1

安装并初始化Hexo

  1. 利用NodeJS中的npm命令即可安装。这里-g是全局的意思（在任意位置点击鼠标右键，选择Git bash）

    npm install -g hexo

  2. Hexo初始化：安装好Hexo之后，我们在想要放置博客的位置，创建一个文件夹，如blog，（在blog文件夹内鼠标右键，选择Git bash）

    hexo init

  3. 至此，全部安装工作已经完成，blog就是你的博客根目录，所有的操作都在里面进行。
  4. 生成静态博客页面（在blog文件夹内鼠标右键，选择Git bash，以后Hexo命令都是这样操作，不再重复说明）

    hexo generate (也可以简写为 hexo g)

  5. 启动本地服务，进行文章预览调试：

    hexo server (也可以简写为 hexo s)

  6. 浏览器输入http://localhost:4000, 就可以看到初始化状态下的Hexo博客静态页,大致样子:

部署到Github中

之前一直说的是本地的服务加载的静态博客，这篇是关于如何将本地静态博客部署至Github中。注意，这里说的是把静态博客（public下的文件）部署到Github，不是根目录下的工程配置文件，工程配置文件里面含有私人各种配置的密钥信息，比如谷歌分析，百度分析的密钥，工程配置文件需要我们本地保存或者上传到私有的代码平台。使用Git备份博客工程文件。

1.github创建仓库

去github创建仓库，前提需要我们注册，仓库名字需要和我们的github账号对应，格式：yourname.github.io，比如我就是sunxinfei.github.io。
这里我没有使用SSH密钥，使用的HTTPS。

2.配置

打开根目录下的_config.yml站点配置文件，对应代码修改为如下，注意空格，注意我这里版本是Hexo3.3.1这个repo链接如下填写

  • deploy:
    type: git
    repo: https://github.com/SunXinFei/sunxinfei.github.io.git
    branch: master

3.上传

如果第一次使用git，执行命令时会弹出登录框来登陆。

  • npm install hexo-deployer-git –save 最新版本Hexo可能不需要执行该命令执行安装deployer插件
  • hexo d 运行该命令即可上传到Github仓库中，这里有的可能会报错，复制对应信息搜下，应该可以解决。

关于Hexo工程文件

在站点根目录下，可以看到几个重要的文件夹：

  • source文件夹
    • _posts文件夹，里面存放每个文章的md首页
    • categorie文件夹，后面自己创建，里面存放分类的md首页
    • tags文件夹，后面自己创建，里面存放标签的md首页
    • CNAME 后面自己创建的，用来指向自定义的域名
  • public文件夹，放置生成的静态文件
  • _config.yml，位于站点根目录下，主要包含 Hexo 本身的配置，称为 站点配置文件

_config.yml站点配置文件

注意！冒号后面的空格，后面配置还会强调
大部分解释在Hexo官方文档中可以看到，这里我主要配置了

  • title: XinFei’s Notes
  • author: XinFei
  • language: zh-Hans
  • per_page: 5
  • theme: next 这个在下一个教程关于Next配置会提到
  • skip_render: README.md 这个可以避免渲染项目中的README.md文件

命令安装插件错误

  • npm WARN optional SKIPPING OPTIONAL DEPENDENCY: fsevents 如果你在安装插件过程中出现这样的错误，是因为 fsevent是mac osx系统的，在win或者Linux下使用了 所以会有警告，忽略即可，出现npm WARN是正常的，只要不出现npm ERR就没事

总结：Hexo常用命令

  • hexo init 新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。
  • hexo new xxxx 创建一篇新文章，创建的md文件会放置在_posts文件夹中
  • hexo generate （简写为hexo g）生成静态文件
  • hexo server （简写为hexo s）启动服务器。默认情况下，访问网址为： http://localhost:4000/
  • hexo deploy （简写为hexo d）部署网站，后面会经常用这个命令来提交到Github上。
  • hexo clean 清除缓存文件 (db.json) 和已生成的静态文件 (public)，这个主要用来删除文章（_posts文件夹下的md文件）之后，清除之前生成的缓存。
  • hexo version 显示 Hexo 版本
